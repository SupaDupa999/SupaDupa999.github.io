  2. Click on the name of the root repository

Files

Uploading a File From computer to Github

At the repository
  1. Click on the + button, then click on Upload a file
  2. Search file on the File Search panel that appears
  3. Mark the file then click Open
  4. When file loaded, click Commit Change

Downloading a File from Github to Computer

At the repository
  1. Click on the name of the file to download
  2. Click on the raw button (button with 3 dots near top right of page
  3. Click Download File
Depending on browser setting, the file will be downloaded
either to the Download folder of your computer
or a download search panel will appear,
allow you to direct which folder to receive the file

Edit a File

At the repository holding the file
  1. Click on the file name to show file
  2. Click on the icon with the pencil (above and to the right of file listing)
  3. In the editing panel, change the file as required
  4. In the filename box, change the file name as required
  5. When editing finished, click the Commit Changes button
If the file is a webpage in the io repository, Call the page on a browser to check results
Please Note: After editing the new page does not appear on www until after 20-40 seconds

Copy a File

  1. Download the file to a temporary folder on your computer
  2. Rename (by Edit) the exiting file name to the new one
  3. Upload the original file from the temporary folder back
  4. In the computer and using file Explorer, delete the temporary file

Move a File to Another Repository

  1. Download the file to a temporary folder on your computer
  2. Upload the file back to another repository
  3. Delete the original file
  4. In the computer and using file Explorer, delete the temporary file

Create a New File in a Repository

Method 1. By Coding
  1. Click on the + button, then select Create a New File
  2. Enter the contents of the page
  3. Enter the name of the page
  4. Click Commit Change button
Method 2. Using a Template
  1. Use a template file in the repository or upload a template from your computer
  2. Edit the template file
    • Change the contents
    • Change the file name
  3. Click Commit Change button
Method 3. Copying another web page
  1. Get the source code
    1. Obtain the back end
      On the browser menu go to Tools->Browser Tools->Page-Source
      The back end should appear in another tab
    2. Mark and coly the whole back end to the clipboard (ctrl+c)
  2. At the respository, click on the + button, then select Create a New File
  3. Paste content of clipboard to the new file (ctrl+v)
  4. Enter name for the new file
  5. Click Commit Change button

Delete a File from a Repository

  1. Click on the file name
  2. Click on the icon with 3 dots at top right corner of page
  3. Select Delete File
  4. Click Commit Changes (twice in different places on page)
Note: a deleted file, along with its editing history, cannot be recovered

Clean a File's edit history

Github keeps a history of changes to a file since its creation.
This makes the file size large and contents confusing

When a file is finalized
Its a good idea to remove the editing history

To do so
  1. Download the file to the temporary folder in the computer
    Only the file code is downloaded, not the history
  2. Delete the file in the repository
    The history will also be deleted
  3. Upload the original file from temporary folder to the repository
    The file is back, but history gone
